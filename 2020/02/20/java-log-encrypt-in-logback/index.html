Java 日志加密处理

发表于 2020-02-20 更新于 2021-08-18

日志中包含了很大敏感的信息，尤其是一些系统的异常日志更加不能随意处理，所以在日志追加到文件时，需要对文件进行加密处理，这既能保护软件自身，又可以保护用户的敏感数据。

在当下流行的日志技术体系 (SLF4J + Logback) 中，应该如何进行加密的操作呢？

默认情况下，logback 会将日志输出到控制台上，并且默认的日志级别为 debug。为了能够更加细致的控制 logback 的日志输出情况，引入 logback 的配置文件。logback 的默认配置文件名称为 logback.xml，放置在项目的 classpath 下即可生效。

文件日志记录全部加密

在 logback 体系中提供了众多的 Appender ，我们可以选择不同的 Appender 来实现不同位置的输出，如：

  • ConsoleAppender 输出到控制台
  • FileAppender 输出到文件中
  • RollingFileAppender 拆分和归档压缩后输出到文件中
  • ……

后记：日志加密的整体思路，就是对日志 Appender 中的字节流进行加密操作，最终进行文档或控制台的追加，扩展的思路都是继承 logback 中的类，并对其中关键的方法进行重写。

Java 当前的日志框架体系也是经过了多年的发展，为了更加了解日志体系，需要系统的了解一下 Java 日志体系的发展。

在编程中加密技术，也经历了数载的发展，也有必要去了解一下加密的发展史。

本文探讨的，需要加密的日志，都是由日志框架采集的，这部分日志是服务运行的日志，供服务的开发者浏览。另外还有一类日志，是需要软件开发中写入数据库中的日志，这部分日志需要给用户展示，为了保证这部分日志的安全，需要进行的不再是加密了，而是签名。签名的目的是为了防止数据的篡改和伪造，后期有机会也可以将这部分日志的设计进行详解。
