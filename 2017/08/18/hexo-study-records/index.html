Hexo学习记录

发表于 2017-08-18 更新于 2021-06-11

Hexo tutorial

安装Node.js、Hexo并配置环境
  1. 安装Node.js ，记得选择add to path选项
  2. 在cmd下，通过node -v 来查看当前Node.js的版本
  3. 通过npm -v 来确认node.js 的组件管理器版本
  4. 将Hexo添加到path
创建博客项目并配置NexT主题
  1. 建立一个myBlog文件夹，从命令行进入这个文件夹。
  2. 运行hexo init 启动项目，运行npm install 安装一些必要的组件
  3. 然后在myBlog 文件夹下面生成博客项目
  4. 其中_config.yml 文件是博客的详细配置信息
  5. hexo g(hexo generate) 生成静态文件
  6. hexo s 开启hexo自带的网络服务器功能(127.0.0.1:4000)
  7. Hexo themes 主题网站
  8. 通过Hexo themes 中每个主题的Git Hub 连接到相应地址下载主题文件，解压后放在myBlog\themes 文件夹中
  9. 在myBlog \ _config.yml 中配置，设置为：theme: next 关于NexT 主题的使用方法，有专业的介绍网址
发布博文
  1. 在cmd命令行输入：hexo new 文章标题 以此来创建一篇博文
  2. 会在myBlog\source\ _posts 文件夹里面生成博文的源文件
  3. 用sublime打开，被三个横线围起来的内容，称为博文的元信息
  4. 源文件是.md的文件，支持Markdown语言。Markdown–入门指南
  5. 保存完毕后，运行hexo g ，然后hexo s 浏览博客网站，便可以看到新添加的博文
  6. hexo g 运行上面程序的一瞬间，hexo会把public删除掉，新建pubic，然后根据source目录下面的_posts 中的文件，生成相应的博文HTML放到新的public文件夹里面。并且，public完全是依靠主题的模板，和source里面的md文件而存在的，一个负责博客的外观，一个负责博客的内容。只要这两部分的内容不发生改变，通过hexo g 新建的public就不会改变
  7. 当然hexo还提供了草稿功能。hexo new draft 文章标题 ，便可以创建一个草稿文件，写完后 publish 文章标题.md 便可推送，继续执行hexo g 便生成最新的文件了
博客设置
  1. 此时，博客网站的页面只有三个，一个是首页，一个是归档，一个是标签。如果想添加一个about页面来显示个人信息，便通过 hexo new page about ，然后在myBlog\source 中便生成一个about文件夹，里面包含了一个index.md文件，可以在元信息下面填入自己个人个性说明
  2. 在myBlog\themes\next\ _config.yml 文件中找到负责配置菜单栏的那一部分配置代码，将about前面的#号去掉，便可以将about页面显示出来。并且，menu下面的顺序，便是博客主页菜单栏的排布顺序
  3. 其他的页面添加方法和上面相似
博客部署
  1. 将博客部署在GitHub，推荐Git的使用教程
  2. 首先在GitHub官网 注册一个自己的账号。然后创建一个新的仓库，New repository，注意Repository name 便是博客的域名 username.github.io，例如我自己的博客 https://utokato.github.io/
  3. 在git 的命令行中，通过SSH来实现自己电脑和GitHub的连接。首先，ssh-keygen 来为电脑生成公匙和私匙，然后在C:\Users\username 下会产生一个 .ssh的文件夹，其中id_rsa是私钥文件，id_rsa.pub是公钥文件。用sublime打开公匙，复制。在GitHub用户Settings的左侧SSH and GPG keys 中New SSH key ，标题用于区别自己的电脑，Key中粘贴公匙中的内容。此时已经完成自己电脑和远程GitHub的连接
  4. 再次打开myBlog\ _config.yml ，末尾位置有deploy的信息，用来配置部署博客用的。repo配置为 git@github.com:username/username.github.io.git。注意：配置文件的每个冒号后面都有一个空格
  5. 配置好以后，在cmd中运行npm install hexo-deployer-git –save(注意save前面是2个短线)，安装hexo需要的部署工具。然后运行hexo d -g ，意思是先generate，再deploy。先在本地生成public文件夹，然后往GitHub仓库推送public文件夹里面的文件
  6. 至此，整个博客已经完全部署在了GitHub上了，别人也可以通过username.github.io来访问你的博客
后记

虽说整个博客也已经全部部署了，但是还是存在很多的细节问题，改善这些可以让博客更加美观和人性化，同时可以提高自己的认识。最后的最后，感谢lier.sapce的作者，参考他的博客，才能完成自己的博客创建和部署
