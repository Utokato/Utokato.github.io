Hexo迁移记录

发表于 2018-06-27 更新于 2021-08-18

我们也许会遇到这样的需求，本来在一台电脑上完美的搭建了Hexo的博客，从环境到部署都完成了，如Utokato’s Notes 。但是，换了一台电脑呢？我们又如何能够继续对该博客进行维护呢？

​ 首先，我们需要考虑的问题是，既然这个博客已经和GitHub上面的某个仓库(Utokato.github.io)进行了绑定，所以新电脑上的Git环境需要与我们自己的GitHub账号绑定。其次，新的电脑中肯定需要搭建Node.js和Hexo的环境。最后，我们自己的文件需要移植到新的电脑。

GitHub环境搭建

​ GitHub，推荐Git的使用教程

​ 在git的命令行中，通过SSH的方式来实现新电脑与自己GitHub账号的关联。

  1. 使用ssh-keygen命令来为新电脑生成公钥和私钥。执行后，在C:\Users\username会产生一个.ssh的文件夹，其中id_rsa是私钥文件，id_rsa.pub是公钥文件。
  2. 使用文本编辑器打开id_rsa.pub，复制全部内容。
  3. 在GitHub用户Settings的左侧SSH and GPG keys中，点击New SSH key。其中，title用于区别不同的电脑环境，在key中粘粘id_rsa.pub中的内容。
  4. 此时，已经完成了新环境与远程GitHub的关联。
Node.js和Hexo环境搭建
  1. 安装Node.js，并添加到环境变量中。
  2. 在cmd中，通过node -v 来查看当前Node.js的版本，npm -v 来确认Node.js 的组件管理器版本。
  3. 同样的，在cmd中执行npm install -g hexo-cli
  4. 将Hexo添加到环境变量中。因为Node.js将Hexo安装到了C:\Users\Username\AppData\Roaming\npm这个文件夹下，所以将这个路径添加到系统的环境变中。(Username是自己的用户名，可以先去路径下面去看下是否已经产生了hexo相关的文件，如:hexo.cmd)
移植本地文件

​ 现在来讨论一下在迁移过程中哪些文件是必须迁移的。

​ 首先，之前自己修的文件，如: _config.yml ，theme文件夹中的主题，以及source里面的自己的博客，这些是必须要拷贝的。此外，scaffolds文件夹定义了文章的模板，package.json说明了使用哪些包，.gitignore 文件限定了提交时哪些文件可以忽略，所以需要拷贝。

​ 其实，这三个文件不是我们修改的，所以即使丢失了，也没有关系，我们可以建立一个新的文件夹，然后在里面执行hexo init，就会生成这三个文件，我们只需要将它们拷贝过来使用即可。

​ 总结来说，_config.yml，theme/，source/，scaffolds/，package.json，.gitignore，是需要拷贝的。

​ 再来讨论哪些文件是不必拷贝的。

​ 首先是.git文件，无论是在站点根目录下，还是主题目录下的.git文件，都可以删掉。然后，node_modules在npm install会重新生成，public会在hexo g时重新生成，.deploy_git文件夹在hexo d是自动生成，db.json文件也可以删除。

​ 其实上面这些文件也就是.gitignore文件里面记载的可以忽略的内容。

​ 总结: .git/，node_modules/，public/，.deploy_git/，db.json文件需要删除。

​ 说到这里，其实可以直接将一个通用的方式更加便捷的方式来完成上面的操作，就是在新的环境中，选择一个新的文件夹，如myBlog，命令行进入该文件夹，执行hexo init，再执行npm install安装必要组件。此时在文件myBlog中生成了一个全新的博客工程。将必须迁移的文件和文件夹替换为老博客工程中的文件，基本上就可以完成操作了。

​ 最后，安装一些必要的组件。

  1. 为了hexo d来部署到GitHub上，需要安装npm install hexo-deployer-git --save

  2. 为了建立RSS订阅，需要安装 npm install hexo-generator-feed --save

  3. 为了建立站点地图，需要安装 npm install hexo-generator-sitemap --save

    至此，迁移工作已经结束了。使用hexo d -g进行生成文件并部署，如果正常的话，就迁移成功了。

总结

​ 大致的迁移工作都已经在上面的正文部分进行了说明，在此感谢知乎用户skycrown，看着他的答案进行了梳理和记录，并完成了迁移。其中可能还有些错误，需要多多查询资料了。
