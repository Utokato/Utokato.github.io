多端同步维护Hexo博客

发表于 2018-06-28 更新于 2021-08-17

​ 上一篇文章中，记录了Hexo迁移的记录，给我们使用新的电脑维护博客提供了思路，同时分析了Hexo的目录结构，告诉我们哪些是必要的资源，哪些是可以通过软件自动生成的。

​ 但是，这个方法还是存在着一些缺陷，比如我们自己的必要资源如何迁移到新的电脑上呢？难道是使用优盘拷贝吗？当然不用，在网上看到这样一个思路，就是使用Git的分支来实现。

思路分析

​ 首先，我们需要知道的是，hexo帮助我们将博客发送到GitHub上时，已经将.md文件转换成了网页文件。另外，注意到hexo目录下的文件结构和GitHub上的文件结构是不同的。GitHub上的文件结构与public文件夹下的文件结构相同。hexo只是将public里面的东西推送到了GitHub上，其余的文件则保留在本地目录中。

​ 所以，我们可以将hexo生成的静态博客文件默认放置在master分支上，即将public里面的资源由master管理；另外建立一个分支，如hexo，将hexo的源文件放置在hexo分支上。更换新端后，直接git clone hexo 分支即可。值得一提的是，hexo源文件有几十兆，并不是所有的文件到要放到这个新的分支中，node_modules目录可以使用npm install命令生成，public 目录可以使用hexo g命令生成，.deploy_git是hexo d命令生成的，所以可以将这三个目录在.gitignore 里忽略提交。

​ 一言蔽之，将hexo必要源文件放置在Git的一个分支上，这样我们就不需要使用优盘拷贝了，不是吗？

分步操作

旧电脑上的操作

  1. 已完全搭建好了自己的博客，并实现了与GitHub的关联，可以参考前面的文章哟。

  2. 在GitHub上对该仓库创建一个新的分支，名字可以自己随便起，如hexo。可以将这个分支设置为默认的分支(不是必须的)。该仓库->Settings->Branches->Default branch中将默认分支设为hexo，保存。

  3. 在电脑上创建一个新的文件夹，将该仓库克隆到这个新文件夹中。git clone -b hexo git@github.com:xxx/xxx.github.io.git。git clone 默认会复制master分支，可以使用 -b hexo指明克隆hexo分支，因为源码都放置了hexo分支中，不需要对master分支进行操作。如果上一步设置了默认分支为hexo，会更方便。

  4. 完成上面操作后，可以在Git bash中执行git branch查看一下当前的分支，应该为hexo分支。

  5. 将本地博客的所有文件拷贝到这个username.github.io目录中。

  6. 提交hexo分支，顺序执行git add .、git commit -m 'backup hexo files to new branch named hexo'、git push ，即可将博客的部署环境提交到GitHub个人仓库的新分支(hexo)中。

  7. 现在，在GitHub的username.github.io仓库中已经能够看到两个分支的差异。master分支和hexo分支各自保存一个版本，master分支用于保存博客静态资源，提供博客页面供人访问；hexo分支用于备份部署文件，供自己更新维护。

新电脑上的操作
  1. 在新的电脑上，首先是环境的搭建，主要是Git、Node.js和hexo环境，需要完成新电脑与GitHub的互联。可参看上一篇文章。
  2. 将hexo分支中的文件clone到新电脑本地的文件夹中。git clone git@github.com:xxx/xxx.github.io.git，默认使用的是hexo分支。
  3. 由于.gitignore定义将一些文件忽略了，所以需要安装一下。分别执行npm install生成node_modules目录，执行hexo g生成博客静态文件(public目录)。
  4. 此时就可以在新的电脑上更新博客了。编辑、撰写文章或其他博客的更新改动结束后，记得将最新的资源同步到hexo分支上，顺序执行git add .、git commit -m 'backup hexo files to new branch named hexo'(引号中是备注，可自由填写)、git push。
  5. 最后，执行hexo d -g进行发布。
日常操作
  1. 切换到不同电脑后，首先要从GitHub上拉取最新的hexo分支内容，git pull ，不写分支名称，会拉取默认分支的信息。
  2. 编写、修改博客，使用hexo d -g命令发布。
  3. 把对文件的最新修改推送到GitHub的hexo分支上，依次执行git add .、git commit -m 'commit message'、git push 。

总结

​ 基本上完成了多端维护hexo博客的操作记录，其中需要对Git的操作有基本了解，不熟悉的话，可以去看廖大大的教程，简洁易懂。

​ 最后，感谢简书上的这篇文章的作者，和这篇文章的作者。
